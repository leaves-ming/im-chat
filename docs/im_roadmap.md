# 自建 IM 系统：21 天落地行动清单

这份清单参考了 `MallChatIM` 的核心架构，将复杂的 IM 系统拆解为 3 个阶段，共 21 天。

## 阶段一：夯实基础（第 1-7 天）
**目标**：搭建 Spring Boot 基础框架，掌握 WebSocket 长连接核心。

*   **Day 1: 环境搭建与单体脚手架**
    *   **目标**：跑通 Spring Boot + MySQL + MyBatis
    *   **知识点**：Maven 多模块管理、MyBatis 配置。
    *   **落地**：创建 `im-server`, `im-common` 模块。
*   **Day 2: Netty 入门与 WebSocket 服务**
    *   **目标**：搭建一个简单的 WebSocket 接口。
    *   **知识点**：Netty 架构(EventLoop, Channel, Pipeline)、WebSocket 协议协议头。
    *   **落地**：编写 `NettyWebSocketServer`，能接收并打印控制台消息。
*   **Day 3: 用户认证与 JWT 核心**
    *   **目标**：实现用户登录与 Token 校验。
    *   **知识点**：JWT 签名、拦截器 (Interceptor) 鉴权。
    *   **落地**：登录接口返回 Token，Netty 握手时通过 URL 或 Header 鉴权。
*   **Day 4: Channel 管理与 UID 绑定**
    *   **目标**：把连接上来的长连接跟用户 ID 对应起来。
    *   **知识点**：`ConcurrentHashMap` 并发安全、Channel 生命周期钩子。
    *   **落地**：实现 `NettyAdapter` 维护 `Map<Long, Channel>`。
*   **Day 5: 消息协议设计 (Protobuf/JSON)**
    *   **目标**：定义消息在网络上传输的格式。
    *   **知识点**：序列化/反序列化选型、应用层协议头设计。
    *   **落地**：定义 `Message` 基础类，包含 `cmd`（指令）、`data`（负载）、`seq`（序列号）。
*   **Day 6: 单聊流程闭环**
    *   **目标**：用户 A 发消息，用户 B 实时收到。
    *   **知识点**：服务端路由分发逻辑。
    *   **落地**：`send` 接口 -> 查 Map 找 B 的 Channel -> `push` 消息。
*   **Day 7: 第一周小结与 Bug 修复**
    *   **目标**：处理断线重连基础逻辑。
    *   **落地**：实现简单的 `heartbeat`（心跳包）机制。

---

## 阶段二：核心功能与可靠性（第 8-14 天）
**目标**：由“能通”向“好用”和“不丢”演进。

*   **Day 8: 数据库表结构深度设计**
    *   **目标**：设计支持高并发的聊天历史表、联系人表。
    *   **知识点**：读写分离思路、索引优化。
    *   **落地**：创建 `message`, `contact`, `group` 等表。
*   **Day 9: 消息可靠性：持久化方案**
    *   **目标**：先落库，后推送。
    *   **知识点**：读写扩散概念（IM 领域核心）。
    *   **落地**：Service 层实现“存储 + 触发推送”逻辑。
*   **Day 10: Spring Event 解耦利器**
    *   **目标**：把“存储”和“推送”做代码解耦。
    *   **知识点**：`ApplicationEvent` 与 `@EventListener`。
    *   **落地**：发送消息后发出 Event，监听器异步执行推送。
*   **Day 11: 群聊功能实现 (1): 成员管理**
    *   **目标**：入群、退群、成员列表。
    *   **知识点**：集合操作、缓存热点成员信息。
*   **Day 12: 群聊功能实现 (2): 消息推送优化**
    *   **目标**：群里发一条消息，100 个人同时收到。
    *   **知识点**：批处理发送、多线程推送。
    *   **落地**：`ChannelGroup` 批量操作。
*   **Day 13: 敏感词过滤系统**
    *   **目标**：参考 MallChat 实现自己的过滤。
    *   **知识点**：DFA (确定有穷自动机) 算法。
    *   **落地**：敏感词树构建及过滤拦截。
*   **Day 14: 图片/文件消息支持**
    *   **目标**：文件上传与消息链接化。
    *   **知识点**：OSS 存储原理、URL 安全。

---

## 阶段三：高性能与架构升级（第 15-21 天）
**目标**：应对高并发，向专业系统靠拢。

*   **Day 15: Redis 引入与在线状态管理**
    *   **目标**：不仅仅是 Map，要能在多台服务器间同步状态。
    *   **知识点**：Redis Hash/Set。
    *   **落地**：用户上线/下线同步 Redis。
*   **Day 16: 引入 RocketMQ：消息异步化**
    *   **目标**：处理流量高峰，消息削峰填谷。
    *   **知识点**：消息队列基础、可靠消息投递。
    *   **落地**：Service 发消息到 MQ，Netty 服务端订阅 MQ 进行推送。
*   **Day 17: 撤回消息与消息阅读状态**
    *   **目标**：实现“消息已读”与“撤回”逻辑。
    *   **知识点**：消息状态机设计、特定指令下发。
*   **Day 18: 分布式下的 WebSocket：消息路由**
    *   **目标**：如果 B 在 Server 2，A 在 Server 1 怎么发给 B？
    *   **知识点**：MQ 广播机制。
    *   **落地**：利用 MQ 广播消息，各个节点自行判断用户是否在线。
*   **Day 19: 性能压测与 Netty 调优**
    *   **目标**：知道你的系统瓶颈在哪里。
    *   **知识点**：JMeter 压测、Netty 参数（SO_BACKLOG等）。
*   **Day 20: 进阶：消息离线拉取策略**
    *   **目标**：用户不在线时怎么补课？
    *   **知识点**：`LastMessageID` 与版本号增量拉取。
*   **Day 21: 最终演练与文档整理**
    *   **目标**：完整交付。

---

## 给您的落地锦囊：
1.  **不要一开始就做分布式**：先在一个 Spring Boot 里把 Netty 跑通。
2.  **死磕协议设计**：IM 的本质是协议，定义好 JSON 或 Protobuf 的格式是重中之重。
3.  **多看 `MallChatIM` 的 `common-common` 包**：那是精华所在，特别是里面的 Event 机制。
